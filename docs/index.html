The "Phantom API" Hazard

When developers ask LLMs to migrate code (e.g., upgrading from Pydantic V1 to V2), the model relies on Parametric Memory. Since models like Llama 3.2 have training cutoffs, they lack "knowledge" of breaking changes introduced recently.

This leads to a specific type of hallucination we call the "Confident Guess": The model invents a method that sounds plausible but does not exist, causing production crashes.

Solution: Non-Parametric Grounding

PatchPilot decouples "Reasoning" from "Knowledge". We use the LLM solely as a reasoning engine, while RAG (Retrieval Augmented Generation) provides the dynamic knowledge base.

  • Targeted Retrieval We inject precise API signatures from official "Migration Guides" directly into the context window.

The Tech Stack

LangGraph FAISS (Vector DB) Llama 3.2

We utilize a Multi-Agent Architecture. A 'Researcher Agent' scrapes documentation, a 'Planner Agent' creates the migration strategy, and a 'Coder Agent' executes it.

System Methodology

PatchPilot employs a sophisticated Agentic Pipeline. The first two stages are fully operational, with a third validation stage currently in prototyping.

1. Contextual Analysis

Operational

The system parses input code into an Abstract Syntax Tree (AST) to identify imports. It then queries a Vector Database (FAISS) for the target version's documentation using semantic search.

2. Plan & Synthesize

Operational

The Planner Agent generates a step-by-step strategy using Chain-of-Thought (CoT) prompting. It enforces a strict constraint: "No code without citation," requiring source IDs for every change.

Verifiable Citations

LIVE
